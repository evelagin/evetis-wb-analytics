# CLAUDE.md

Ты работаешь над проектом EVETIS WB Analytics.

## Контекст проекта

EVETIS WB Analytics — это система управленческой отчетности для бренда EVETIS на Wildberries.

Технологии:
- Google Sheets
- Google Apps Script
- Wildberries API
- ручная загрузка финансовых Excel-отчетов WB
- справочники SKU, себестоимости и наборов

Цель проекта:
Получить простую, надежную и проверяемую систему, которая показывает:
- заказы,
- продажи,
- возвраты,
- выплаты WB,
- комиссии,
- логистику,
- хранение,
- удержания,
- рекламу,
- себестоимость,
- прибыль,
- маржинальность,
- остатки,
- скорость продаж,
- рекомендации по поставкам.

## Главный принцип

Не усложнять.

Любое изменение должно быть:
1. минимальным,
2. понятным,
3. проверяемым,
4. обратимым.

## Язык проекта

Все названия листов, заголовки, статусы, сообщения, комментарии и пользовательские элементы в Google Sheets должны быть на русском языке.

Английские технические имена допустимы только внутри кода.

## Что запрещено

Запрещено:
- создавать новые листы без явного разрешения;
- переписывать весь проект без отдельного согласования;
- добавлять патчи поверх патчей;
- менять PNL_TOTAL без контрольных сумм;
- смешивать RAW-данные и расчетные данные на одном листе;
- использовать рекламу WB как точную поартикульную себестоимость, если источник не дает связку SKU → расход;
- удалять функции без объяснения;
- менять структуру колонок без описания в CHANGELOG.md.

## Приоритет источников данных

1. RAW-данные WB.
2. Финансовые Excel-отчеты WB.
3. SKU_MASTER.
4. COST_HISTORY.
5. BUNDLES.
6. Расчетные листы.

## Перед написанием кода

Перед любым кодом сначала:
1. опиши, что именно меняешь;
2. укажи, какие листы затрагиваются;
3. укажи риск поломки;
4. предложи контрольные цифры для проверки;
5. только потом предлагай код.

## Формат ответа

Всегда отвечай структурно:

1. Что обнаружено.
2. Почему это проблема.
3. Что предлагается изменить.
4. Какие файлы/листы будут затронуты.
5. Как проверить результат.
