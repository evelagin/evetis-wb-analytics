# EVETIS WB Analytics — передача дел (сессия 2026-06-29)

Продолжение работы в новом чате Cowork. Память проекта (`MEMORY.md`) подхватится автоматически. Этот документ фиксирует, на каком мы этапе и что делать дальше.

## Главное решение сессии: архитектура
Остаёмся на **Google Sheets** (без сервера/доп. затрат, роли пока не нужны). Цель — довести скелет до ума: данные сами тянутся по триггерам, корректно раскладываются, сверху наглядный дашборд. Современный визуал делаем **через Apps Script web-app** (HTML-страница в браузере, читает данные из листов; прообраз — `ads-dashboard.html`), а не формулами на листе. План — `docs/roadmap_sheets_v2.md` (5 фаз).

## Ключевые факты по данным (подтверждены в этой сессии)
- **Заказы/продажи (Статистика API)** хранятся у WB **только 90 дней** от даты продажи. Глубже API не отдаёт. Сентябрь 2024 по заказам недоступен; операционку копим вперёд.
- **История денег с сен 2024 ДОСТУПНА** через новый финансовый API (см. ниже).
- Сторонние сервисы видят историю, потому что копят её у себя с момента подключения — не потому что WB отдаёт назад.

## Финансовый API (новый, категория «Финансы») — рабочий
- Хост `https://finance-api.wildberries.ru`, токен категории **«Финансы»** в Script Property **`WB_TOKEN_FINANCE`**.
- `POST /api/finance/v1/sales-reports/list` — тело `{dateFrom, dateTo}` (обязательны). Вернул **189 недельных отчётов, глубина 2024-09-02 … 2026-06-21**. В каждой записи готовые недельные итоги: `forPaySum, retailAmountSum, deliveryServiceSum, paidStorageSum, deductionSum, penaltySum, paidAcceptanceSum, avgSalePercent` и др.
- `POST /api/finance/v1/sales-reports/detailed/{reportId}` — тело `{rrdid, limit}`, пагинация по `rrdId` (как старый rrdid). Отдаёт строки в т.ч. за сен 2024.
- **Суммы приходят строкой** ("45.34") — парсить в число. **Логистика-деньги = `deliveryService`** (а `deliveryAmount` — счётчик доставок, не путать). Комиссия WB = `vw` (+`vwNds`), эквайринг = `acquiringFee`, хранение = `paidStorage`, приёмка = `paidAcceptance`, штраф = `penalty`, удержание = `deduction`, к перечислению = `forPay`, выручка = `retailAmount`, классификатор операции = `sellerOperName`, тип документа = `docTypeName`.
- На одну неделю часто **два `reportId`** (основной + корректировка) → дедуп строго по безопасному ключу **`reportId|rrdId`**, не по периоду.
- **Старый `GET /api/v5/supplier/reportDetailByPeriod` отключают 15.07.2026** — это и есть причина миграции.

## Где мы в коде (Фаза 0)
Создан файл **`apps-script/WbFinanceApiV1.gs`** (новый, в репозитории). Содержит:
- Разведка (чтение/логи): `showWbFinanceV1ReportsList`, `wbFinanceV1DetailedSample`.
- Загрузчик: `wbFinV1AdaptRow_` (новые camelCase → старые имена `FINANCE_API_FIELD_MAP_` + парсинг сумм), `wbFinV1FetchDetailedAll_` (пагинация), `wbFinV1ImportReport_` (single-report тест с replace-slice по `report_id`), `wbFinanceV1ImportOneReportTest`.
- Бэкфилл (резюмируемый, ИСПРАВЛЕН): `wbFinanceV1Backfill`, `wbFinanceV1BackfillStatus`, `wbFinanceV1BackfillReset`, `wbFinV1BuildSeenRrdSet_`, `wbFinanceV1CheckDuplicates`.
- Метка наших строк в RAW: `source_api = WB_API_FIN_V1`. Правила безопасности: строго токен `WB_TOKEN_FINANCE`; запись запрещена без колонок `source_api`/`report_id`; `row_hash = MD5("WB_API_FIN_V1|reportId|rrdId")`; HTTP 204 = штатное завершение пагинации.

### Что проверено
- Импорт ОДНОГО отчёта (757272781, 15–21.06.2026): detailed-суммы совпали с `list`-итогами **точь-в-точь** (forPay 4396.37, retail 4849.01, логистика 943.99), UNMAPPED нет. Маппинг/парсинг/запись — корректны.

### Что сломалось и как починено
- Первый вариант бэкфилла делал per-report `deleteRow`-очистку по всему листу → на разросшемся RAW: `IllegalStateException` и «Exceeded maximum execution time» (один большой отчёт >6 мин). Декабрь–февраль дают по 1000–3800 строк/неделю (похоже на реальный объём).
- **Бэкфилл переписан**: один скан листа за прогон, дедуп по безопасному ключу `reportId|rrdId`, запись только новых строк, без очистки по периоду. Падения и риск задвоения при повторных/прерванных запусках устранены.

## Процесс/окружение (важно)
- Claude правит файлы в локальной папке = origin/main (`github.com/evelagin/evetis-wb-analytics`). Евгений: GPT-аудит → GitHub Desktop (публикация) → **вручную копирует код в редактор Apps Script** (Chrome). Перед кодом — мини-ТЗ; каждую правку — в `CHANGELOG.md`.
- **Claude in Chrome НЕ может управлять Google Docs/Sheets** (заблокировано политикой) — поэтому загрузчики запускает Евгений из редактора Apps Script, Claude читает результат по логам/через коннектор Drive.
- Готового MCP-коннектора GitHub в реестре нет; запись в Sheets/деплой Apps Script коннекторами недоступны. Чтение живой таблицы — через коннектор Google Drive (прямой gviz через web_fetch висит по таймауту).
