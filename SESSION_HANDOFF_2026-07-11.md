# SESSION HANDOFF — 2026-07-11

Снимок состояния проекта EVETIS WB Analytics на конец сессии 11.07.2026.
Главное событие сессии: **миграция хранилища из Google Sheets в BigQuery** (упёрлись в лимит 10 млн ячеек книги).

---

## 1. Что произошло и почему

При backfill финансов Apps Script упал с ошибкой «количество ячеек превысит 10 000 000». Причина — лимит книги Google Sheets на весь файл. RAW_WB_FINANCE = 69 колонок, полная история ~12 млн ячеек в одном листе. Вывод: Sheets как хранилище исчерпан.

**Решение:** переносим хранилище в **BigQuery** (free tier 10 ГБ + 1 ТБ запросов/мес; Google-нативно; Apps Script имеет встроенный сервис BigQuery — вся логика тяги WB API сохранена, меняется только «слив»).

**Целевая архитектура:** Apps Script (тяга WB API) → BigQuery (RAW_* + дедуп-вью) → SQL-витрины → Looker Studio (постоянный дашборд). Тонкая Google-таблица останется пультом/справочником.

### ✅ Формула комиссии подтверждена на реальных строках
- **База комиссии = `retail_price_withdisc_rub × quantity`** (цена ДО SPP), НЕ `retail_amount`.
- `retail_amount` = сколько заплатил клиент (после SPP); скидку SPP финансирует WB.
- `for_pay = база − комиссия(commission_percent% × база) − acquiring_fee` (сходится до копейки).
- **Чистая комиссия % = (база − for_pay − acquiring) / база** ≈ нативный `commission_percent`.
- Комиссия РОСЛА: ~22,5% (2024-09) → ~28% (лето-2025) → 32–34,5% (2026). НЕ константа.
- ⚠️ `commission_amount` (ppvz_vw) не использовать; `retail_amount − for_pay` (gross) неверно (SPP).

**Рычаги прибыли по величине:** комиссия → удержание/реклама → логистика.
