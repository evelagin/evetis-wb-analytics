# PROJECT_RULES.md

## Правила проекта EVETIS WB Analytics

### 1. Архитектура

Проект делится на 5 слоев:

1. RAW-слой — сырые данные.
2. CLEAN-слой — очищенные и нормализованные данные.
3. Справочники — SKU, себестоимость, наборы.
4. PNL-слой — расчет прибыли.
5. Dashboard-слой — управленческие выводы.

### 2. RAW-слой

RAW-листы нельзя использовать для ручных формул.

RAW-листы:
- RAW_WB_ORDERS
- RAW_WB_SALES_RETURNS
- RAW_WB_FINANCE
- RAW_WB_ADS
- RAW_WB_STOCKS

### 3. Справочники

Справочники:
- SKU_MASTER
- COST_HISTORY
- BUNDLES
- SETTINGS

### 4. Финансы

RAW_WB_FINANCE является главным источником финансового факта WB.

PNL_TOTAL должен сверяться с:
- продажами,
- возвратами,
- выплатами WB,
- логистикой,
- хранением,
- удержаниями,
- эквайрингом,
- рекламой,
- себестоимостью,
- прибылью.

### 5. Реклама

Рекламные расходы WB делятся на два типа:

1. Факт удержаний из финансового отчета WB.
2. Статистика рекламных кампаний из WB API.

Если нет точной связки расхода с SKU, нельзя показывать расход рекламы по SKU как факт.

Можно показывать:
- общий расход рекламы по кабинету;
- аналитику рекламных кампаний;
- оценочное распределение только с явной пометкой "оценка".

### 6. Изменения

Каждое изменение должно фиксироваться в CHANGELOG.md.

Формат:
- дата,
- что изменено,
- зачем изменено,
- какие листы затронуты,
- как проверить.
